# Правила Регистрации Компаний И Менеджеров

## 1. Базовый принцип

В системе нет открытой самостоятельной регистрации управляющих компаний и сотрудников.

Регистрация идет сверху вниз:

1. создатель платформы вручную добавляет компанию
2. создатель платформы привязывает компанию к Telegram
3. создатель платформы выдает компании временный пароль
4. компания входит через Telegram по кнопке и пароль
5. после первого входа компания меняет пароль
6. создатель платформы создает первого руководителя компании или назначает его в карточке компании
7. руководитель компании уже сам создает сотрудников внутри своей учетной записи

## 2. Нормализация сущности компании

В системе `компания` это не роль и не сотрудник.

`Компания` это отдельная бизнес-сущность, которая идет по старшинству сразу после создателя платформы.

У компании есть собственные:

- реквизиты
- Telegram-привязка
- пароль входа
- сотрудники
- объекты
- клиенты
- платежи
- документы

Первый пользователь внутри компании это не сама компания, а `руководитель компании`.

Техническая роль руководителя компании:

- `company_admin`

## 3. Кто может создавать компанию

Только создатель платформы.

Он вручную заполняет:

- название компании
- внутренний код компании
- Telegram ID компании или подтвержденного владельца компании
- Telegram username, если есть
- временный пароль
- статус компании

Самостоятельная регистрация компании через форму не допускается.

## 4. Как компания входит в систему

Для компании вход идет по схеме `Telegram + пароль`.

Поток такой:

1. создатель платформы вносит Telegram ID компании
2. представитель компании пишет в Telegram-бот
3. backend сверяет Telegram ID с карточкой компании
4. если Telegram совпал, бот выдает ссылку в кабинет компании
5. компания вводит временный пароль
6. при первом входе система требует сменить пароль
7. после этого открывается кабинет компании

## 5. Первый пользователь компании

После создания компании создатель платформы назначает первого руководителя компании.

Это главный пользователь внутри компании.

Ему доступны:

- вход в кабинет своей компании
- просмотр только своей компании
- создание сотрудников своей компании
- назначение ролей сотрудникам
- блокировка сотрудников своей компании

Он не может:

- видеть другие компании
- создавать новые компании
- менять создателя платформы

## 6. Кто такой руководитель компании

Руководитель компании это главный администратор внутри конкретной компании.

Он управляет только своим контуром:

- менеджеры
- поддержка
- бухгалтерия
- объекты
- клиенты

По сути уровни разделяются так:

- создатель платформы управляет компаниями
- компания является отдельным контуром данных
- руководитель компании управляет сотрудниками своей компании

## 7. Как создаются сотрудники компании

Новых менеджеров и других сотрудников не создает создатель платформы, если компания уже активирована.

Их создает руководитель внутри своей компании.

При создании сотрудника руководитель указывает:

- ФИО
- роль
- телефон
- Telegram ID
- Telegram username, если есть
- статус

После этого сотрудник входит по своей модели доступа внутри компании.

## 8. Правила ролей внутри компании

Для сотрудников компании действуют роли:

- `company_admin`
- `manager`

Роль `company_admin` это руководитель компании.

## 9. Ограничения доступа

### Создатель платформы

Может:

- создавать компании
- привязывать Telegram компании
- выдавать временный пароль компании
- создавать первого руководителя компании
- блокировать любую компанию

### Компания

Как сущность хранит и открывает свой контур данных, но не действует вне своего контура.

### Руководитель компании

Может:

- создавать менеджеров и других сотрудников своей компании
- редактировать сотрудников своей компании
- деактивировать сотрудников своей компании
- видеть все данные своей компании

### Менеджер

Может:

- работать с объектами и клиентами своей компании
- видеть платежи и заявки своей компании по своим правам

Не может:

- создавать компании
- создавать руководителей других компаний
- выходить за пределы своей компании

## 10. Что нужно закрепить в данных

Для такой модели в данных должны быть явные связи:

- `management_companies`
- `management_companies.telegram_id`
- `management_companies.telegram_username`
- `management_companies.temp_password`
- `management_companies.must_change_password`
- `staff.management_company_id`
- `staff.role`
- `staff.telegram_id`
- `staff.telegram_username`
- `staff.status`

Также у сотрудников должен быть статус:

- `active`
- `invited`
- `blocked`

## 11. MVP-правило

Для MVP работаем по простому сценарию:

1. создатель платформы вручную создает компанию
2. выдает ей Telegram-привязку и временный пароль
3. компания входит через Telegram и меняет пароль
4. после этого руководитель компании сам добавляет остальных сотрудников

Это дает понятную и безопасную модель без публичной регистрации.
