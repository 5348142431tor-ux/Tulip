# Правила Регистрации Компаний И Менеджеров

## 1. Базовый принцип

В системе нет открытой самостоятельной регистрации управляющих компаний и сотрудников.

Регистрация идет сверху вниз:

1. создатель платформы вручную добавляет управляющую компанию
2. создатель платформы привязывает компанию к Telegram
3. вход компании подтверждается только через Telegram
4. создатель платформы создает первого управляющего компании
5. управляющий компании уже сам создает менеджеров внутри своей учетной записи

## 2. Кто может создавать компанию

Только создатель платформы.

Он вручную заполняет:

- название компании
- внутренний код компании
- Telegram ID компании или подтвержденного владельца компании
- Telegram username, если есть
- статус компании

Самостоятельная регистрация компании через форму не допускается.

## 3. Как компания входит в систему

Для компании вход только через подтверждение в Telegram.

Поток такой:

1. создатель платформы вносит Telegram ID компании
2. представитель компании пишет в Telegram-бот
3. backend сверяет Telegram ID с карточкой компании
4. если Telegram совпал, система разрешает вход
5. если Telegram не совпал, вход запрещен

Парольный вход для компании не используется.

## 4. Первый пользователь компании

После создания компании создатель платформы вручную создает первого управляющего.

Это главный пользователь компании.

Ему доступны:

- вход в кабинет своей компании
- просмотр только своей компании
- создание сотрудников своей компании
- назначение ролей сотрудникам
- блокировка сотрудников своей компании

Он не может:

- видеть другие компании
- создавать новые компании
- менять владельца платформы

## 5. Кто такой управляющий

Управляющий это главный администратор внутри конкретной управляющей компании.

Он управляет только своим контуром:

- менеджеры
- поддержка
- бухгалтерия
- объекты
- клиенты

По сути роли разделяются так:

- создатель платформы управляет компаниями
- управляющий управляет сотрудниками своей компании

## 6. Как создаются менеджеры

Новых менеджеров не создает создатель платформы, если компания уже активирована.

Их создает управляющий внутри своей компании.

При создании менеджера управляющий указывает:

- ФИО
- роль
- телефон
- Telegram ID
- Telegram username, если есть
- статус

После этого менеджер тоже входит только через Telegram-подтверждение.

## 7. Правила входа сотрудников

Для всех сотрудников компании действует одно правило:

- вход без пароля
- вход только через Telegram
- доступ только в рамках своей компании

После подтверждения Telegram сотрудник получает свою роль:

- manager
- support
- accountant
- company_admin

Роль `company_admin` это и есть управляющий компании.

## 8. Ограничения доступа

### Создатель платформы

Может:

- создавать компании
- привязывать Telegram компании
- создавать первого управляющего
- блокировать любую компанию

### Управляющий компании

Может:

- создавать менеджеров и других сотрудников своей компании
- редактировать сотрудников своей компании
- деактивировать сотрудников своей компании
- видеть все данные своей компании

### Менеджер

Может:

- работать с объектами и клиентами своей компании
- видеть платежи и заявки своей компании по своим правам

Не может:

- создавать компании
- создавать управляющих других компаний
- выходить за пределы своей компании

## 9. Что нужно закрепить в данных

Для такой модели в данных должны быть явные связи:

- `management_companies`
- `staff.management_company_id`
- `staff.role`
- `staff.telegram_id`
- `staff.telegram_username`
- `management_companies.telegram_id`
- `management_companies.telegram_username`

Также у сотрудников должен быть статус:

- `active`
- `invited`
- `blocked`

## 10. MVP-правило

Для MVP работаем по простому сценарию:

1. владелец платформы вручную создает компанию
2. вручную создает первого управляющего
3. компания и сотрудники входят только через Telegram
4. управляющий сам добавляет остальных менеджеров

Это дает понятную и безопасную модель без публичной регистрации и без паролей.
