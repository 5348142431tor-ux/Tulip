# MVP Roadmap

## Этап 1. Фундамент

- определить роли и сценарии
- утвердить структуру PostgreSQL
- создать backend API
- настроить Telegram-бота

## Этап 2. Клиентский контур

- авторизация через Telegram
- экран моих объектов
- экран платежей
- создание заявки
- уведомления по заявкам

## Этап 3. Кабинет УК

- логин для сотрудников
- список клиентов
- список объектов
- таблица платежей
- обработка заявок

## Этап 4. Операционная зрелость

- журнал действий
- шаблоны уведомлений
- фильтры и отчеты
- выгрузки

## Рекомендуемый порядок разработки

1. backend + PostgreSQL
2. Telegram-бот
3. кабинет УК
4. кабинет клиента

Такой порядок дает быстрый результат, потому что Telegram-бот проще всего вывести в реальную работу.
